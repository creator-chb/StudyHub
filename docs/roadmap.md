# StudyHub 项目发展规划

## 📋 总体目标

将 StudyHub 从一个纯前端的轻量级工具，逐步演进为一个功能完整的前后端分离的学习管理平台。

---

## 🎯 发展阶段

### 阶段一：优化纯前端版本（2026 年 3 月 - 5 月）

**目标：** 在保持现有架构的基础上，从"体验完善、工程化、可扩展性"三个维度持续优化

#### 已完成功能（v1.1.0）：

- [x] **模态弹窗交互**：链接管理与任务管理改为模态卡片对话框，支持 ESC/遮罩关闭
- [x] **链接展示优化**：响应式网格卡片布局，超长文字省略处理，空状态提示
- [x] **任务展示优化**：关联链接 Chip 预览，逾期/紧急任务视觉高亮，空状态提示
- [x] **表单验证增强**：URL 格式校验，删除确认对话框
- [x] **安全性改进**：XSS 防护（escapeHtml），统一编辑/新建弹窗复用

---

#### 第一批：用户体验优先（性价比高）✅ 已完成

**版本：v1.2.0 | 发布时间：2026-03-19**

1. **通知系统升级** 🔔
   - [x] Toast/Snackbar 替代 alert（成功 2s，错误 3s）
   - [x] 确认型 Dialog 替代 confirm（支持 ESC/遮罩关闭）
   - [x] 统一提示文案与持续时间规范

2. **表单校验增强** ✅
   - [x] URL 输入实时校验（失焦即校验），显示具体错误原因
   - [x] 任务截止时间禁止选择过去时间（或二次确认）
   - [x] 必填项即时红字提示，不只在提交时报错

3. **任务状态可视化** 📊
   - [x] 任务完成状态（复选标记）
   - [x] 已完成项目自动置灰或归档到"已完成"分区
   - [x] 动态显示"剩余时间"（如：剩余 2 天 3 小时）
   - [x] 逾期后持续显示"已逾期 X 天"

### 阶段二：设计与原型（2026 年 4 月 - 5 月）

**目标：** 完成 v2.0 的技术选型和架构设计

#### 技术栈调研与确定：

1. **前端框架对比**
   - Vue.js 3 vs React
   - 构建工具：Vite vs Webpack
   - UI 组件库选择

2. **后端技术选型**
   - Node.js + Express/Fastify
   - Python + FastAPI/Django
   - Go + Gin（备选）

3. **数据库选择**
   - MongoDB（文档型，灵活）
   - MySQL/PostgreSQL（关系型，规范）
   - SQLite（轻量级，初期使用）

4. **认证方案**
   - JWT Token
   - Session/Cookie
   - OAuth 2.0（第三方登录）

- [ ] 技术栈确定文档

- [ ] 数据库 ER 图设计

## 💡 开发建议

1. **小步快跑** - 每个阶段专注一个目标，不要急于求成
2. **文档先行** - 写代码前先设计，减少返工
3. **测试保障** - 关键功能必须写测试
4. **用户反馈** - 及时收集意见，调整方向
5. **代码复用** - v1.0 的业务逻辑可以迁移到 v2.0
