## 🚀 v2.0 - 全栈架构演进（规划中）

**目标版本**: v2.0.0  
**预计周期**: 6-8 周  
**技术栈**: Node.js + Express + PostgreSQL + Redis

### 核心理念

采用**渐进式演进**策略，在保持现有功能稳定的基础上，通过存储抽象层实现前后端解耦，最终达成完整的前后端分离架构。

### 阶段规划

#### Phase 0: 架构基础重构（第 1 周）✅

**目标**: 建立存储抽象层，为后端集成做准备

**任务清单**:
- [x] **存储接口抽象化**
  - 创建 `frontend/src/js/modules/storage/` 目录
  - 定义 `AbstractStorage` 接口规范
  - 重构现有 `Storage` 模块为 `LocalStorageAdapter`
  - 实现配置开关 `features.backendSync`

- [x] **项目结构调整**
  - 创建 `frontend/` 目录，迁移现有前端代码
  - 创建 `backend/` 目录，初始化 Node.js 项目
  - 创建 `shared/` 目录，存放共享类型定义
  - 更新构建配置和开发脚本

- [x] **HTML 文件结构规范化**
  - 创建 `legacy/` 目录归档历史版本
  - 归档 `studyhub.html` 到 `legacy/studyhub_v1_legacy.html`
  - 明确区分主入口文件和开发版本入口
  - 为主入口和归档文件添加详细注释说明

- [x] **文档更新**
  - 更新 README 开发指引
  - 记录架构决策（ADR-001）

**交付物**:
- ✅ 可运行的前端版本（功能不变，路径调整为 `frontend/`）
- ✅ 后端项目基础框架（`backend/`，Phase 0 骨架）
- ✅ 存储抽象层实现（AbstractStorage + LocalStorageAdapter）
- ✅ 清晰的 HTML 入口文件结构（主入口/开发版/历史版本）

#### 入口文件说明

| 入口文件 | 用途 | 使用场景 |
|---------|------|---------|
| `/index.html` | **主应用入口** | 生产环境，包含完整认证系统 |
| `/frontend/index.html` | 前端开发版本 | 独立前端开发/测试，无后端依赖 |
| `/legacy/studyhub_v1_legacy.html` | v1.0 历史版本 | 需要独立HTML文件的场景 |

#### 技术栈详情

| 层级 | 技术选型 | 说明 |
|-----|---------|------|
| 前端 | 原生 JS + CSS3 | 保持现有技术栈，零依赖 |
| 后端 | Node.js + Express + TypeScript | 成熟稳定，生态丰富 |
| 数据库 | PostgreSQL 15 | 关系型数据库，支持 JSON |
| 缓存 | Redis 7 | Session、热点数据缓存 |
| 部署 | Docker + Docker Compose | 容器化部署 |
| CI/CD | GitHub Actions | 自动化流水线 |
