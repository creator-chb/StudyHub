# StudyHub 项目发展规划

## 📋 总体目标

将 StudyHub 从一个纯前端的轻量级工具，逐步演进为一个功能完整的前后端分离的学习管理平台。

---

## 🎯 发展阶段

### 阶段一：优化纯前端版本（2026 年 3 月 - 4 月）

**目标：** 在保持现有架构的基础上，完善用户体验和功能完整性

#### 已完成功能（v1.1.0）：

- [x] **模态弹窗交互**：链接管理与任务管理改为模态卡片对话框，支持 ESC/遮罩关闭
- [x] **链接展示优化**：响应式网格卡片布局，超长文字省略处理，空状态提示
- [x] **任务展示优化**：关联链接 Chip 预览，逾期/紧急任务视觉高亮，空状态提示
- [x] **表单验证增强**：URL 格式校验，删除确认对话框
- [x] **安全性改进**：XSS 防护（escapeHtml），统一编辑/新建弹窗复用

#### 待开发功能：

1. **数据管理增强**
   - [ ] 数据导入/导出（JSON 格式）
   - [ ] 数据备份提醒
   - [ ] 批量操作功能

2. **任务管理优化**
   - [ ] 任务分类/标签系统
   - [ ] 任务优先级设置
   - [ ] 任务完成状态追踪
   - [ ] 重复任务支持

3. **用户体验改进**
   - [ ] 搜索和过滤功能
   - [ ] 主题切换（深色/浅色模式）
   - [ ] 自定义颜色配置
   - [ ] 响应式布局优化（移动端适配）

4. **界面优化**
   - [ ] 拖拽排序功能
   - [ ] 改进通知提示（Toast 替代 alert）

#### 技术债务清理：
- [ ] 代码模块化重构
- [ ] 添加 JSDoc 注释
- [ ] 优化事件处理逻辑
- [ ] 性能优化

### 阶段二：设计与原型（2026 年 4 月 - 5 月）

**目标：** 完成 v2.0 的技术选型和架构设计

#### 技术栈调研与确定：

1. **前端框架对比**
   - Vue.js 3 vs React
   - 构建工具：Vite vs Webpack
   - UI 组件库选择

2. **后端技术选型**
   - Node.js + Express/Fastify
   - Python + FastAPI/Django
   - Go + Gin（备选）

3. **数据库选择**
   - MongoDB（文档型，灵活）
   - MySQL/PostgreSQL（关系型，规范）
   - SQLite（轻量级，初期使用）

4. **认证方案**
   - JWT Token
   - Session/Cookie
   - OAuth 2.0（第三方登录）

#### 产出物：
- [ ] 技术栈确定文档
- [ ] API 接口设计规范（RESTful）
- [ ] 数据库 ER 图设计
- [ ] 系统架构图
- [ ] 前端组件设计稿

## 💡 开发建议

1. **小步快跑** - 每个阶段专注一个目标，不要急于求成
2. **文档先行** - 写代码前先设计，减少返工
3. **测试保障** - 关键功能必须写测试
4. **用户反馈** - 及时收集意见，调整方向
5. **代码复用** - v1.0 的业务逻辑可以迁移到 v2.0
